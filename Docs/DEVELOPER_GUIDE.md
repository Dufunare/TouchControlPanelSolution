# TouchControlPanel 项目开发者指南

## 一、项目背景与目标

本项目旨在为 OpenHaptics 触觉设备开发一套“长期可维护、易扩展、团队友好”的 Qt 控制面板。目标不仅是“能跑起来”，更是让后续功能增长、多人协作、模块复用都变得轻松。

- **面向对象**：支持后端设备逻辑与前端 UI 解耦。
- **分层架构**：设备采集、控制调度、界面展示各自独立。
- **可扩展性**：便于后续加新业务、拆新模块、接入新设备。
- **团队协作**：让不同成员可以专注于自己负责的层，不互相干扰。

## 二、架构设计理念

### 1. 解耦思想

- **后端与前端彻底分离**：后端只暴露最小接口，前端无需关心 OpenHaptics 细节。
- **数据流单向**：设备状态通过快照流向前端，前端通过控制器发起动作。
- **每层只做本职**：设备采集不碰 UI，UI 不碰设备 API，控制器只做调度。

### 2. 分层结构

- **设备后端层（ControlBackend）**：负责设备初始化、采样、状态维护。
- **控制层（DeviceController）**：负责调度后端、轮询状态、转发信号。
- **界面层（MainWindow + Widgets）**：负责布局、展示、交互。

### 3. 可扩展性

- 新功能优先拆分为独立 Widget/Controller/Service。
- 共享数据结构统一放在 shared/，便于多端共用。
- 视频、轨迹、参数等大块业务建议单独建模块，不要堆进主窗口。

## 三、各层职责与协作方式

| 层级       | 主要职责                     | 典型文件/类                 |
| ---------- | ---------------------------- | --------------------------- |
| 设备后端层 | 设备初始化、采样、状态快照   | ControlBackend/TouchBackend |
| 控制层     | 调度后端、轮询状态、信号转发 | DeviceController            |
| 界面层     | 布局、展示、交互、信号连接   | MainWindow、各类 Widget     |
| 共享数据层 | 前后端共享状态结构           | shared/DeviceState.h        |

- **主窗口只做拼装和信号连线，不写业务逻辑。**
- **每个 Widget 只负责自己的显示和交互。**
- **后端只暴露最小接口，前端不直接 include OpenHaptics。**

## 四、典型开发流程与协作建议

### 1. 新功能开发流程

1. 明确功能属于哪一层（设备/控制/界面/共享）。
2. 新建或扩展对应的模块/Widget/Controller。
3. 只在必要的地方修改共享数据结构。
4. 主窗口只做装配和信号连线。
5. 编写/更新文档，说明新模块的职责和用法。

### 2. 协作建议

- **多人协作时，优先分工到不同层或不同 Widget。**
- **接口变更要同步更新文档和头文件注释。**
- **每次合并前自查：是否有跨层耦合、主窗口变臃肿、共享数据被滥用。**
- **遇到新需求，先问“能否独立成新模块”，不要一股脑堆进现有类。**

## 五、常见开发误区与最佳实践

### 误区

- 把设备循环写进 UI 线程（会卡死界面）。
- 主窗口承担过多业务逻辑。
- 所有新功能都往 DeviceState 里塞。
- 控件跨线程直接操作（应通过信号/queued invoke）。

### 最佳实践

- 设备高频采样，前端低频轮询（当前 8ms）。
- 复杂业务优先拆分为独立 Controller/Widget。
- 共享数据结构只放“小而高频”的状态。
- 大数据（如视频帧）单独通道传递，不进 DeviceState。
- 重要接口、数据结构、信号都写清楚注释。

## 六、新成员快速上手指引

1. 通读本指南和 ARCHITECTURE.md，理解分层思想。
2. 跑通现有工程，熟悉主窗口、控制器、后端的协作。
3. 阅读 FILE_PURPOSES.md，了解每个文件的职责。
4. 按 HOW_TO_EXTEND.md 指引尝试加一个小功能。
5. 有问题优先查文档和注释，再向团队提问。

## 七、FAQ

- **Q: 为什么不用 Qt 的多线程直接采集设备？**
  A: OpenHaptics 官方推荐用 scheduler 回调，线程安全且高效。
- **Q: 为什么主窗口不直接 include 设备头文件？**
  A: 为了解耦和可维护性，所有设备细节都封装在后端。
- **Q: 新业务怎么加？**
  A: 先拆分为独立模块，主窗口只做拼装和信号连线。
